# 五库联调防洪四预技能

本技能执行防洪四预（生成调度方案→预演→预案）全流程业务，针对五库（陆浑、故县、三门峡、小浪底、河口村）联调场景。

## ⚠️ 重要限制说明

目前**仅支持针对2021年秋汛（10月2日至10月7日）** 的调度方案生成。其他时间因未接入五库调度模型，暂时无法生成调度方案。

## 核心工具

| 类别       | 工具                              | 用途                     |
| -------- | ------------------------------- | ---------------------- |
| **调度方案** | `generate_dispatch_scheme`      | 生成五库联调调度方案单（唯一一套，基于2021年秋汛真实调度数据） |
| **预演指令** | `send_simulation_command`       | 向前端发送预演指令（异步发送，不等待响应）  |
| **触发执行** | `trigger_simulation_execution` | 向前端发送触发预演执行的指令 |
| **数据查询** | `get_reservoir_latest_realtime` | 查询水库最新实时数据               |
| **数据查询** | `get_reservoir_realtime`        | 查询水库指定时间的历史数据            |
| **数据查询** | `get_river_latest_realtime`     | 查询水文站最新实时数据              |
| **预案生成** | `load_plan_template` | 自动查询数据并生成完整的洪水调度预案 |
| **文档发送** | `send_plan_document_url` | 将预案文档URL发送给前端展示 |

## 四预流程

### 用户输入示例

> "模拟21年秋汛，进行五库调度"

⚠️ **注意**：目前仅支持针对2021年秋汛（10月2日至10月7日）的调度方案生成。`generate_dispatch_scheme` 工具只接受 `start_time` 参数，生成的方案即基于2021年秋汛**唯一的一套真实调度数据**，不涉及额外约束条件设置。

***

### 第一步：查询五库水情并获取调度方案

**重要提示**：目前仅支持针对2021年秋汛（10月2日至10月7日）的调度方案生成。其他时间因未接入五库调度模型，暂时无法生成调度方案。

**操作步骤**：

1. **查询五库水情并获取调度方案**
   - **判断时间需求**：首先识别用户需求中的时间信息，仅支持2021年秋汛（10月2日至10月7日）。
   - **获取数据**：
     - 调用 `get_reservoir_realtime(time="2021-10-02")` 获取五库2021年秋汛历史数据
     - 调用 `get_river_realtime(time="2021-10-02")` 获取水文站2021年秋汛历史数据
     - 涉及水文站：龙门镇、白马寺、黑石关、花园口等
   - **总结汇报水情**：向用户汇报五库水情（水位、库容、出入库流量等关键指标），说明是2021年秋汛历史数据
   - **获取调度方案**：调用 `generate_dispatch_scheme(start_time="2021-10-02")` 获取唯一的一套调度方案（基于2021年秋汛真实调度数据，工具只接受start_time参数，不含其他约束条件）

### 第二步：执行预演

用户确认调度方案后执行预演。

**操作步骤**：

1. **向前端发送预演指令**
   - 调用 `send_simulation_command(scheme_id=...)`（传入方案ID，如 DS-0001；不传则由前端选择）
   - **注意**：此工具为异步发送，调用后立即返回确认信息，不等待前端执行结果
2. **返回确认信息**
   - 返回格式：`{"success": true, "task_id": "sim_xxxx", "message": "预演指令已发送"}`
3. **提示用户并描述调度方案**
   - 告知用户预演指令已发送，任务处于准备完成状态
   - **描述所选调度方案单内容**：
     - 方案名称和ID
     - 调度时段（开始时间、结束时间、总时长）
     - 五库调度要点（各水库水位变化范围、出入库流量变化趋势等）

***

### 第三步：等待用户触发预演或生成预案

用户确认调度方案后，需要选择触发预演执行或直接生成预案。

**操作步骤**：

1. **询问用户选择**
   - 提示用户："预演任务已准备就绪，您可以选择：1) 触发预演执行 2) 直接生成预案"
2. **根据用户选择执行**
   - **触发预演执行**：调用 `trigger_simulation_execution(task_id=...)`（传入前一步返回的 task_id）
     - 返回格式：`{"success": true, "task_id": "sim_xxxx", "message": "预演执行指令已发送"}`
     - 告知用户预演正在执行，可通过界面查看进度
   - **直接生成预案**：跳过预演执行，直接进入第四步生成预案

调度方案单格式示例：

```json
{
  "success": true,
  "count": 1,
  "schemes": [
    {
      "scheme_id": "DS-0001",
      "scheme_name": "2021年秋汛调度方案",
      "start_date": "2021-10-02",
      "end_date": "2021-10-07",
      "reservoirs": {
        "三门峡水库": { ... },
        "小浪底水库": { ... },
        "陆浑水库": { ... },
        "故县水库": { ... },
        "河口村水库": { ... }
      },
      "hydrological_stations": {
        "龙门镇": { ... },
        "白马寺": { ... },
        "黑石关": { ... },
        "花园口": { ... }
      }
    }
  ]
}
```

***

### 第四步：生成预案

用户确认预演结果后生成预案。

**操作步骤**：

1. **获取基础预案数据**
   - 调用 `load_plan_template(generation_time="YYYY-MM-DD", scheme_id="DS-XXXX")` 获取预案模板
   - 参数说明：
     - `scheme_id`：调度方案ID（如 DS-0001），传入后将使用调度方案中的数据计算（各水库最大值和水文站数据）
     - `generation_time`：可选，预演任务的开始时间
   - 该工具会自动查询水库和水文站数据、判断预警等级、生成调度方案并返回结构化数据
   - **注意**：返回的模板内容中包含【大模型需...】格式的占位符，表示需要大模型进一步补充
2. **大模型分析完善**
   - 根据获取的基础数据，由大模型进行深度分析
   - 对于包含【大模型需...】占位符的章节，大模型需调用相应工具获取数据进行补充
   - 严格按照模板章节完善模板
   - 生成完整的洪水调度预案分析报告
3. **保存预案文档**
   - 将完善后的预案内容通过文件操作工具保存到本地文件
4. **发送文件URL给前端**
   - 调用 `send_plan_document_url(document_url="文件URL", document_name="预案文件名")`
   - 前端收到后会在界面中展示或提供下载链接

***

## 工具参数详细说明

### 调度方案类工具

| 工具名 | 参数 | 参数说明 | 是否必填 | 默认值 |
|-------|------|---------|---------|--------|
| `generate_dispatch_scheme` | `start_time` | 调度开始时间（格式：YYYY-MM-DD），仅限 2021-10-02 至 2021-10-07 范围内有效 | 否 | null（使用当前时间） |

> **注意**：`generate_dispatch_scheme` 仅接受 `start_time` 一个参数，生成的方案基于2021年秋汛唯一一套真实调度数据。不再支持其他约束条件参数。

### 预演指令类工具

| 工具名 | 参数 | 参数说明 | 是否必填 |
|--------|------|---------|---------|
| `send_simulation_command` | `scheme_id` | 调度方案ID（如 DS-0001） | 否 |
| `trigger_simulation_execution` | `task_id` | 预演任务ID（由 send_simulation_command 返回） | 是 |

### 数据查询类工具

| 工具名 | 参数 | 参数说明 | 是否必填 |
|--------|------|---------|---------|
| `get_reservoir_latest_realtime` | 无参数 | 查询所有水库最新实时数据 | - |
| `get_reservoir_realtime` | `reservoir`, `start_date`, `end_date` | 查询水库指定时间的历史数据 | 是 |
| `get_river_latest_realtime` | 无参数 | 查询所有水文站最新实时数据 | - |
| `list_realtime_hydrology` | `station`, `start_date`, `end_date` | 查询水文站指定时间的历史数据 | 是 |

### 预案生成类工具

| 工具名 | 参数 | 参数说明 | 是否必填 |
|--------|------|---------|---------|
| `load_plan_template` | `generation_time` | 预案生成时间，不指定则使用当前时间 | 否 |
| | `scheme_id` | 调度方案ID（如 DS-0001），传入后使用调度方案中各水库最大值和水文站数据计算 | 否 |

### 文档发送类工具

| 工具名 | 参数 | 参数说明 | 是否必填 |
|--------|------|---------|---------|
| `send_plan_document_url` | `document_url` | 预案文档的访问URL | 是 |
| | `document_name` | 预案文档名称（用于前端显示） | 是 |

## 交互规则

1. **每步必确认**：生成调度方案、预演、预案每步完成后必须等待用户确认
2. **默认重新启动**：当用户触发"模拟21年秋汛，进行五库调度"类指令时，如未明确说明"继续之前未完成的任务"，则**默认作为全新流程启动**，不承接历史残留的预演任务ID、调度方案编号等进度。每次启动都应重新生成方案编号。
3. **方案唯一性**：`generate_dispatch_scheme` 仅接受 `start_time` 参数，生成的方案即基于2021年秋汛真实调度数据的唯一一套方案。不再支持用户指定约束条件（如花园口流量控制、各水库水位限制等），也不再有"有约束/无约束方案"之分。
