# 五库联调防洪四预技能

本技能执行防洪四预（生成调度方案→预演→预案）全流程业务，针对五库（陆浑、故县、三门峡、小浪底、河口村）联调场景。

## 五库概况

| 水库  | 编码                       | 位置    | 主要功能    |
| --- | ------------------------ | ----- | ------- |
| 陆浑  | LuHun (BDA80200721)      | 伊洛河   | 蓄洪、滞洪   |
| 故县  | GuXian (BDA80000661)     | 洛河    | 蓄洪、发电   |
| 三门峡 | SanMenXia (BDA00000111)  | 黄河干流  | 防凌、蓄洪   |
| 小浪底 | XiaoLangDi (BDA00000121) | 黄河干流  | 调水调沙、蓄洪 |
| 河口村 | HeKouCun (BDA00000761)   | 黄河最下游 | 蓄洪、滞洪   |

## 核心工具

| 类别       | 工具                              | 用途                     |
| -------- | ------------------------------- | ---------------------- |
| **调度方案** | `generate_dispatch_scheme`      | 生成五库联调调度方案单（支持多水库独立约束） |
| **预演指令** | `send_simulation_command`       | 向前端发送预演指令（异步发送，不等待响应）  |
| **数据查询** | `get_reservoir_latest_realtime` | 查询水库最新实时数据               |
| **数据查询** | `get_reservoir_realtime`        | 查询水库指定时间的历史数据            |
| **数据查询** | `get_river_latest_realtime`     | 查询水文站最新实时数据              |
| **数据查询** | `get_river_realtime`            | 查询水文站指定时间的历史数据           |

\| **预案生成** | `load_plan_template` | 自动查询实时数据并生成完整的洪水调度预案 |
\| **预演指令** | `trigger_simulation_execution` | 向前端发送触发预演执行的指令 |
\| **文档发送** | `send_plan_document_url` | 将预案文档URL发送给前端展示 |

## 四预流程

### 用户输入示例

> "控制花园口流量不超过4500m³/s，且调整小浪底水位不超248m，怎样进行五库联调？"

用户会指定约束条件，如：

- 某个水文站的流量限制（如花园口）
- 某个水库的水位限制（如小浪底≤248m）
- 某个水库的库容限制
- 是否需要汛前腾空库容

***

### 第一步：查询五库水情并确认需求

**操作步骤**：

1. **查询五库水情**
   - **判断时间需求**：首先识别用户需求中的时间信息。若用户提到历史事件（如"21年秋汛，即21年10月2日到10月7日"），则需要查询对应历史时间点的数据；若无时间要求，则查询最新实时数据。
   - **获取数据**：
     - 历史场景：调用 `get_reservoir_realtime(time="YYYY-MM-DD")` 获取五库历史数据，调用 `get_river_realtime(time="YYYY-MM-DD")` 获取水文站历史数据
     - 当前场景：调用 `get_reservoir_latest_realtime()` 获取五库最新数据，调用 `get_river_latest_realtime()` 获取水文站最新数据
     - 涉及水文站：龙门镇、白马寺、黑石关、花园口等
   - **总结汇报**：向用户汇报五库水情（水位、库容、出入库流量等关键指标），明确说明是历史数据还是最新数据
   - **方案数量**：等待用户确认生成几套调度方案
   - **约束条件**：等待用户确认是否有约束要求，包括：
     - 是否需要控制花园口流量？如果是，最大流量限制是多少？
     - 是否需要控制各水库水位？如果是，各水库的水位上限是多少？
     - 是否需要控制各水库库容？如果是，各水库的库容上限是多少？
     - 是否需要汛前腾空库容？如果是，目标库容比例是多少？
     - 是否需要指定调度开始时间？

### 第二步：生成调度方案单

**操作步骤**：

1. **生成调度方案单**
   - 根据用户需求调用 `generate_dispatch_scheme()`
   - 支持的约束参数：
     - 花园口流量控制：`control_huayuankou_flow=true`, `huayuankou_max_flow=4500`
     - 汛前腾空库容：`flood_season_empty_storage=true`, `target_storage_ratio=0.7`
     - 各水库水位约束：`smx_max_level`, `xld_max_level`, `lh_max_level`, `gx_max_level`, `hkc_max_level`
     - 各水库库容约束：`smx_max_storage`, `xld_max_storage`, `lh_max_storage`, `gx_max_storage`, `hkc_max_storage`
     - 调度开始时间：`start_time="2024-07-15"`
2. **返回调度方案单给用户**

调度方案单格式示例：

```json
{
  "success": true,
  "count": 3,
  "constraints_applied": {
    "control_huayuankou_flow": true,
    "huayuankou_max_flow": 4500,
    "flood_season_empty_storage": false,
    "reservoir_constraints": {
      "小浪底水库": {"max_level": 248, "max_storage": null}
    }
  },
  "schemes": [
    {
      "scheme_id": "DS-0001",
      "scheme_name": "2024年汛期调度方案1",
      "description": "第1套汛期洪水调度方案，涵盖主要水库群联合调度",
      "basin": "黄河",
      "start_date": "2024-07-15",
      "end_date": "2024-07-22",
      "constraints": ["花园口流量控制: ≤ 4500 m³/s"],
      "details": [
        {
          "time": 1778047200000,
          "reservoirs": [
            {
              "reservoir_name": "小浪底水库",
              "station_code": "BDA00000121",
              "water_level": 247.85,
              "storage": 126.5,
              "inflow": 1500.0,
              "outflow": 1450.0
            }
          ],
          "total_inflow": 5000.0,
          "total_outflow": 4800.0,
          "huayuankou_flow": 4080.0
        }
      ]
    }
  ]
}
```

1. **总结方案并等待用户确认**
   - 向用户对比各调度方案内容和预期效果概述（如预计花园口流量、各水库水位等）,等待用户选择哪一套方案进行预演，以及是否需要调整约束条件重新生成

***

### 第三步：执行预演

用户确认调度方案后执行预演。

**操作步骤**：

1. **向前端发送预演指令**
   - 调用 `send_simulation_command(scheme_id=...)`（传入方案ID，如 DS-0001；不传则由前端选择）
   - **注意**：此工具为异步发送，调用后立即返回确认信息，不等待前端执行结果
2. **返回确认信息**
   - 返回格式：`{"success": true, "task_id": "sim_xxxx", "message": "预演指令已发送"}`
3. **提示用户并描述调度方案**
   - 告知用户预演指令已发送，任务处于准备完成状态
   - **描述所选调度方案单内容**：
     - 方案名称和ID
     - 方案约束条件（如花园口流量限制、水库水位限制等）
     - 调度时段（开始时间、结束时间、总时长）
     - 五库调度要点（各水库目标水位、出库流量等）

***

### 第四步：等待用户触发预演或生成预案

用户确认调度方案后，需要选择触发预演执行或直接生成预案。

**操作步骤**：

1. **询问用户选择**
   - 提示用户："预演任务已准备就绪，您可以选择：1) 触发预演执行 2) 直接生成预案"
2. **根据用户选择执行**
   - **触发预演执行**：调用 `trigger_simulation_execution(task_id=...)`（传入第三步返回的 task\_id）
     - 返回格式：`{"success": true, "task_id": "sim_xxxx", "message": "预演执行指令已发送"}`
     - 告知用户预演正在执行，可通过界面查看进度
   - **直接生成预案**：跳过预演执行，直接进入第五步生成预案

***

### 第五步：生成预案

用户确认预演结果后生成预案。

**操作步骤**：

1. **获取基础预案数据**
   - 调用 `load_plan_template(generation_time="YYYY-MM-DD", scheme_id="DS-XXXX")` 获取预案模板
   - 参数说明：
     - `generation_time`：预演任务的开始时间，用于查询对应时间点的历史水情数据
     - `scheme_id`：调度方案ID（如 DS-0001），传入后将使用调度方案中的数据计算，否则查询实时/历史数据
   - 该工具会根据指定时间自动查询水库和水文站数据、判断预警等级、生成调度方案并返回结构化数据
   - **调度方案场景**：传入 `scheme_id` 参数时，使用调度方案中的各水库最大值（水位、入库流量、出库流量、库容）和水文站数据进行计算
   - **历史场景**：若用户询问历史事件（如"21年秋汛"），传入对应历史时间，工具会查询该时间点的历史数据
   - **当前场景**：若不传参数或传入当前时间，工具会查询最新实时数据
   - **注意**：返回的模板内容中包含【大模型需...】格式的占位符，表示需要大模型进一步补充
2. **大模型分析完善**
   - 根据获取的基础数据，由大模型进行深度分析
   - 对于包含【大模型需...】占位符的章节，大模型需调用相应工具获取数据进行补充
   - 严格按照模板章节完善模板，有内容的章节需要参考并整理章节内容，需要补充的章节内容调用工具获取或参考知识库相关知识
   - 生成完整的洪水调度预案分析报告
3. **保存预案文档**
   - 将完善后的预案内容通过文件操作工具保存到本地文件并生成文件访问URL
4. **发送文件URL给前端**
   - 调用 `send_plan_document_url(document_url="文件URL", document_name="预案文件名")`
   - 前端收到后会在界面中展示或提供下载链接

***

## 交互规则

1. **每步必确认**：生成调度方案、预演、预案每步完成后必须等待用户确认
2. **预演指令发送**：只需调用 `send_simulation_command`，具体预演由前端实现
3. **异步发送**：`send_simulation_command` 为异步发送，不等待响应，立即返回任务ID
4. **位置名称**：使用英文编码：SanMenXia(三门峡)、XiaoLangDi(小浪底)、LuHun(陆浑)、GuXian(故县)、HeKouCun(河口村)
5. **session\_id**：所有工具的session\_id参数均为可选，由DeerFlow拦截器自动从上下文注入，无需用户输入
