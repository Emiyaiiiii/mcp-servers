# 五库联调防洪四预技能

本技能执行防洪四预（生成调度方案→预演→预案）全流程业务，针对五库（陆浑、故县、三门峡、小浪底、河口村）联调场景。

## ⚠️ 重要说明

**输入数据（Q_Inputsd/Q_Inputxd）** 和 **控制参数（Dispatch_Par）** 是两套独立的东西：
- 输入数据 → 从水文局预报 API 动态获取，用 `get_hydrology_forecast_data` 写入
- 控制参数 → 从参数模板 Excel 应用，用 `apply_parameter_template` 写入

生成调度方案前，必须确保两者都已就绪。

## 场景分类：实时调度 vs 历史模拟

本系统支持两种场景的洪水调度：

### 场景A：实时调度（默认）
- 用户说"调算洪水"、「模拟当前洪水」等，默认走**水文局实时预报数据**
- 调用 `get_hydrology_forecast_plans(date_time=今日)` 获取最新预报方案
- 用户不需要确认数据来源（参见用户画像）

### 场景B：历史模拟 / 演练
- 用户说"模拟XX年秋汛"、"演练洪水"、"回放历史洪水"等
- 需要主动搜索历史日期的预报方案，**不是问用户数据来源，而是搜不同日期直到找到方案**
- 常见历史场景：
  - 2021年秋汛 → 搜索 2021-10-05（方案名含"2021秋汛水文局预报方案"）
- 搜索技巧：从用户提到的大致时间点开始，前后各试几个日期，因为历史方案通常注册在某个特定日期而非整个汛期

### 场景识别规则
| 用户说法 | 场景 | 操作 |
|---------|------|------|
| "调算洪水"、"生成调度方案"、"调算洪水结果" | 实时调度（A） | 默认今日，无需询问数据来源 |
| "模拟21年秋汛"、"模拟XX年"、"演练"、"回放"、"历史" | 历史模拟（B） | 搜索对应历史日期的预报方案 |

## 核心工具

| 类别       | 工具                              | 用途                     |
| -------- | ------------------------------- | ---------------------- |
| **数据获取** | `get_hydrology_forecast_plans`  | 获取水文局预报方案列表 |
| **数据获取** | `get_hydrology_forecast_data`   | 获取方案数据并写入 Q_Inputsd/Q_Inputxd |
| **参数控制** | `apply_parameter_template`      | 应用参数模板（写入 Dispatch_Par） |
| **调度方案** | `generate_dispatch_scheme`      | 生成五库联调调度方案（基于动态数据） |
| **预演指令** | `send_simulation_command`       | 向前端发送预演指令（异步发送，不等待响应）  |
| **触发执行** | `trigger_simulation_execution` | 向前端发送触发预演执行的指令 |
| **数据查询** | `get_reservoir_latest_realtime` | 查询水库最新实时数据               |
| **数据查询** | `get_reservoir_realtime`        | 查询水库指定时间的历史数据            |
| **数据查询** | `get_river_latest_realtime`     | 查询水文站最新实时数据              |
| **预案生成** | `load_plan_template` | 自动查询数据并生成完整的洪水调度预案 |
| **文档发送** | `send_plan_document_url` | 将预案文档URL发送给前端展示 |

## 四预流程

### 用户输入示例

> "根据下大常规方案控制，用五库模型调算洪水结果"
> "用五库联调生成调度方案"
> "进行防洪四预流程"

⚠️ **注意**：当用户说"生成调度方案"、"调算洪水结果"等，流程为：先获取水文局动态数据，再应用参数模板，最后生成方案。

***

### 第一步：获取输入数据 + 应用参数模板 + 生成调度方案

**前置检查**：先检查数据库 Q_Inputsd/Q_Inputxd 是否已有数据。调用 `generate_dispatch_scheme` 会返回数据状态。

**如果数据库为空（无数据）**：

1. **引导用户提供时间**
   - Agent 询问："请提供调度时间，格式如 2025-06-16"
   - 用户："2025-06-16"

2. **获取预报方案列表**
   - 调用 `get_hydrology_forecast_plans(date_time="2025-06-16")`
   - 展示方案列表给用户选择

3. **用户选择方案后获取数据**
   - 调用 `get_hydrology_forecast_data(plcd="方案名", time="时间")`
   - 数据写入 Q_Inputsd 和 Q_Inputxd 表

4. **应用参数模板（注意洪水类型陷阱 + 模板分类选择）**
   - **展示可用模板列表**：调用 `list_parameter_templates()` 展示所有模板给用户选择
   - **词义映射**：用户说"常规调度"时 → 优先选择 `下大洪水控制/方案一`（全五库含特征值sheet）
   - 如果用户说"根据XX方案控制"，直接匹配对应模板
   - 调用 `apply_parameter_template(template_name="模板名", generate_scheme=false)`
   - 仅写入 Dispatch_Par 表，不导入输入数据
   - **⚠️ 陷阱检查（只适用于上大洪水控制的模板）**：应用模板后，需检查洪水类型参数（stcd=46）是否与场景匹配
     - 调用 `modify_dispatch_param(action="list")` 查看所有参数
     - **下大洪水控制/方案X → stcd=46已=1（下大洪水），无需修正 ✅**
     - **上大洪水控制/方案X → stcd=46默认=0（上大洪水）**。但如果用于下大洪水场景（如21年秋汛），需要手动修正：`modify_dispatch_param(action="update", station_name="洪水类型", param_desc="0为上大洪水，1为下大洪水", new_value=1)`
     - 判断是否需要修正的依据：看用户的要求是上大洪水还是下大洪水场景，而不是看模板存哪个目录下
   - **模板sheet完整性提示**：`上大洪水控制`下的模板只含"三门峡、小浪底、下游"3个sheet；`下大洪水控制`下的模板含"特征值、三门峡、小浪底、陆浑、故县、河口村、花园口以上洪水、下游"8个sheet。推荐优先使用下大洪水控制模板。当用户说"五库联调"时，应优先选择包含所有五库的模板。

5. **生成调度方案**
   - 调用 `generate_dispatch_scheme()`
   - 读取 Q_Inputsd/Q_Inputxd（输入数据）+ Dispatch_Par（控制参数）
   - 运行 RegualDispacth.exe 计算
   - 返回各水库计算结果

6. **查询水情并总结汇报**
   - 调用 `get_reservoir_latest_realtime()` 获取五库最新水情
   - 调用 `get_river_latest_realtime()` 获取水文站最新水情
   - 涉及水文站：龙门镇、白马寺、黑石关、花园口等
   - 向用户汇报水情和调度方案计算结果

**如果数据库已有数据**：

1. 跳过数据获取步骤（步骤 1-3），直接进入步骤 4（应用参数模板）
2. 如果用户指定了参数方案，直接应用；否则询问用户
3. 生成调度方案并汇报结果

### 第二步：执行预演

用户确认调度方案后执行预演。

**操作步骤**：

1. **向前端发送预演指令**
   - 调用 `send_simulation_command(scheme_id=...)`（传入方案ID，如 DS-0001；不传则由前端选择）
   - **注意**：此工具为异步发送，调用后立即返回确认信息，不等待前端执行结果
2. **返回确认信息**
   - 返回格式：`{"success": true, "task_id": "sim_xxxx", "message": "预演指令已发送"}`
3. **提示用户并描述调度方案**
   - 告知用户预演指令已发送，任务处于准备完成状态
   - **描述所选调度方案单内容**：
     - 方案名称和ID
     - 调度时段（开始时间、结束时间、总时长）
     - 五库调度要点（各水库水位变化范围、出入库流量变化趋势等）

***

### 第三步：等待用户触发预演或生成预案

用户确认调度方案后，需要选择触发预演执行或直接生成预案。

**操作步骤**：

1. **询问用户选择**
   - 提示用户："预演任务已准备就绪，您可以选择：1) 触发预演执行 2) 直接生成预案"
2. **根据用户选择执行**
   - **触发预演执行**：调用 `trigger_simulation_execution(task_id=...)`（传入前一步返回的 task_id）
     - 返回格式：`{"success": true, "task_id": "sim_xxxx", "message": "预演执行指令已发送"}`
     - 告知用户预演正在执行，可通过界面查看进度
   - **直接生成预案**：跳过预演执行，直接进入第四步生成预案

调度方案单格式示例：

```json
{
  "success": true,
  "count": 1,
  "schemes": [
    {
      "scheme_id": "DS-0001",
      "scheme_name": "五库联调调度方案",
      "start_date": "2025-06-06",
      "end_date": "2025-06-21",
      "reservoirs": {
        "三门峡水库": { ... },
        "小浪底水库": { ... },
        "陆浑水库": { ... },
        "故县水库": { ... },
        "河口村水库": { ... }
      },
      "hydrological_stations": {
        "龙门镇": { ... },
        "白马寺": { ... },
        "黑石关": { ... },
        "花园口": { ... }
      }
    }
  ]
}
```

### 第四步：生成预案

用户确认预演结果后生成预案。

**操作步骤**：

1. **获取基础预案数据**
   - 调用 `load_plan_template(generation_time="YYYY-MM-DD", scheme_id="DS-XXXX")` 获取预案模板
   - 参数说明：
     - `generation_time`：调度方案的**开始时间**（必传），即预案对应调度时段的起始时刻。格式为 "yyyy-MM-dd HH:mm:ss" 或 "yyyy-MM-dd"
     - `scheme_id`：调度方案ID（如 DS-0001），传入后将使用调度方案中的数据计算（各水库最大值和水文站数据）
   - 该工具会自动查询水库和水文站数据、判断预警等级、生成调度方案并返回结构化数据
   - **注意**：返回的模板内容中包含【大模型需...】格式的占位符，表示需要大模型进一步补充
2. **大模型分析完善 — 具体做法**
   - 获取实时补充数据：
     - 调用 `get_reservoir_latest_realtime()` 获取水库最新水情（用于"水情实况"章节）
     - 调用 `get_river_latest_realtime()` 获取河道水文站最新流量（用于"水情实况"章节）
   - 利用 `generate_dispatch_scheme` 返回结果中的 `reservoir_stats` 和 `all_stations` 数据填充调度结果表格
   - 构建完整预案的常见章节结构：
     - 基础信息（方案编号、场景、调度策略、依据文件）
     - 实时雨水工情（初始条件、核心调度参数表）
     - 洪水预报（五库调度结果表、下游演进表）
     - 水库群调度方案（各水库分述：调度方式、关键数据、分析）
     - 预警与风险评估（总体应急响应、各库预警、花园口出险）
     - 调度结果分析（效果表、与控制标准对比、方案间对比）
     - 处置建议（分条列出建议措施）
   - 生成格式：输出为**完整自包含HTML文件**（含封面页、样式、表格），保存到 `/mnt/user-data/outputs/` 目录下
   - 命名规范：`{场景名}五库联调调度预案-{方案ID}.html`
3. **保存预案文档**
   - 通过 `write_file` 将完整HTML内容保存到 `/mnt/user-data/outputs/` 目录下
4. **发送文件URL给前端**
   - 调用 `send_plan_document_url(document_url="文件绝对路径", document_name="预案文件名.html")`

### ⚡ 方案切换（不换数据源，只换参数模板）

当用户要求"切换到方案X重算"时：

1. **展示目标模板参数** — 调用 `show_parameter_template(template_name="...")` 让用户预览参数
2. **应用新模板** — 调用 `apply_parameter_template(template_name="...", generate_scheme=false)` 
   - 这只会覆盖 Dispatch_Par 表，不会影响 Q_Inputsd/Q_Inputxd 中的预报数据
3. **直接重算** — 调用 `generate_dispatch_scheme()`，无需重新获取水文局数据
4. **方案对比汇报** — 新旧方案结果并列展示（五库对比表 + 下游站对比表 + 差异分析），让用户直观看到切换效果

## 交互规则

1. **每步必确认**：数据获取、参数模板选择、调度方案生成、预演、预案每步完成后必须等待用户确认
2. **数据优先**：生成调度方案前，必须确保 Q_Inputsd/Q_Inputxd 有数据。如无数据，引导用户走水文局 API 获取流程
3. **参数独立**：`apply_parameter_template` 只写 Dispatch_Par 表，不导入输入数据。输入数据始终通过 `get_hydrology_forecast_data` 从水文局 API 获取
4. **默认重新启动**：当用户触发"生成调度方案"类指令时，如未明确说明"继续之前未完成的任务"，则**默认作为全新流程启动**
5. **输入数据时间范围**：数据时间范围取决于水文局 API 返回的预报方案（minYmdh ~ maxYmdh）
