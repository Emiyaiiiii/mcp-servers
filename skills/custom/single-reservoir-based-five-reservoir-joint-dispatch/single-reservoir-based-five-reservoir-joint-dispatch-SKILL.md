# 基于单库模型的黄河五库联调方案生成技能

本技能实现：在只有单库调度模型的条件下，生成满足任意约束（如小浪底水位限制）的黄河五库（陆浑、故县、河口村、三门峡、小浪底）联合调度方案。

## 五库地理位置与流向关系（正确版）

根据黄河流域实际地理分布：

| 位置 | 河流 | 水库 | 说明 |
|------|------|------|------|
| 上游 | 黄河干流 | 三门峡 | 通常保持敞泄运用，出库≈入库 |
| 支流（三门峡-小浪底区间） | 伊河 | 陆浑 | 洪水在三门峡下游汇入黄河 |
| 支流（三门峡-小浪底区间） | 洛河 | 故县 | 洪水在三门峡下游汇入黄河 |
| 支流（三门峡-小浪底区间） | 沁河 | 河口村 | 洪水在三门峡下游汇入黄河 |
| 下游 | 黄河干流 | 小浪底 | 黄河干流最后一道控制性水库，承担核心调蓄任务 |

**正确流向（上游→下游）**:
```
黄河上游来水
    ↓
三门峡（敞泄运用）
    ↓ + 伊河（陆浑）+ 洛河（故县）+ 沁河（河口村）支流出库
小浪底水库（承担调蓄，满足水位约束）
    ↓
花园口断面
```

## 三门峡水库运用特点
- 汛期通常保持**敞泄状态**，闸门全开，不拦蓄洪水
- 出库流量 ≈ 入库流量（忽略库区损失）
- 主要功能是调水调沙、控制潼关高程，而非防洪拦蓄

## 核心算法：分级计算+反向迭代调整

### 总体计算框架

```
Step 1: 新安江模型 → 各支流预报入流
Step 2: 并行单库计算 → 三支流水库出库过程
Step 3: 合成小浪底入库 → 考虑三门峡敞泄特性
Step 4: 单库计算小浪底 → 满足用户指定水位约束
Step 5: 约束检查 → 若不满足，向上游支流分配额外拦蓄任务
Step 6: 迭代调整 → 重复Step 2~Step 5 直到收敛
Step 7: 输出完整五库调度方案单
```

## 详细计算步骤

### 第一步：新安江模型产汇流计算

对伊河、洛河、沁河三个支流流域分别运行新安江三水源模型：

**输入**: 各流域逐时段预报降雨量  
**输出**:
- 陆浑水库入库流量 `Q_in_LH(t)`
- 故县水库入库流量 `Q_in_GX(t)`
- 河口村水库入库流量 `Q_in_HKC(t)`

*说明: 三门峡以上黄河干流来水由上游水文站预报提供，无需新安江计算*

---

### 第二步：支流水库单库调度计算（并行）

分别对陆浑、故县、河口村调用单库调度模型：

**输入**:
- 初始水位 `Z0`（从实时水情获取）
- 预报入库过程 `Q_in(t)`
- 约束条件：`Z(t) ≤ 汛期汛限水位`

**输出**:
- 逐时段水位 `Z(t)`
- 逐时段出库流量 `Q_out(t)`
- 逐时段蓄水量 `V(t)`

**计算结果**:
- 伊河出口: 陆浑出库 `Q_out_LH(t)`
- 洛河出口: 故县出库 `Q_out_GX(t)`
- 沁河出口: 河口村出库 `Q_out_HKC(t)`

---

### 第三步：合成小浪底入库流量

由于三门峡保持敞泄运用，出库 ≈ 上游入库，直接合成：

```
Q_in_XLD(t) = 
  Q_SMX_upstream(t)    // 黄河三门峡上游来水预报
+ Q_out_LH(t)          // 伊河陆浑水库出库
+ Q_out_GX(t)          // 洛河故县水库出库  
+ Q_out_HKC(t)         // 沁河河口村水库出库
+ Q_interval(t)        // 三门峡-小浪底区间无控制面积径流
```

---

### 第四步：小浪底水库单库调度计算

**输入**:
- 初始水位 `Z0_XLD`（当前实时水位）
- 合成入库 `Q_in_XLD(t)`
- 用户约束条件: `Z_XLD(t) ≤ Z_limit`（如 248.0 m）

**调用单库调度模型**

**输出**:
- 小浪底逐时段出库 `Q_out_XLD(t)`
- 小浪底逐时段水位 `Z_XLD(t)`
- 小浪底逐时段蓄水量 `V_XLD(t)`

---

### 第五步：约束检查与反向回溯调整

#### 1. 约束检查
检查是否满足所有约束：
- `max(Z_XLD(t)) ≤ Z_limit`（用户指定小浪底水位上限）
- 各支流水库水位 ≤ 自身汛限水位

#### 2. 不满足约束时的调整策略

**调整原则**: 小浪底水位超约束，需要支流额外拦蓄。三门峡敞泄不能拦蓄，全部额外拦蓄任务分配给三个支流。

计算超水量：
```
ΔV = V_XLD_max - V_XLD_limit
```

按支流防洪库容比例分配额外拦蓄任务：
```
总防洪库容 = capFld_LH + capFld_GX + capFld_HKC

ΔV_LH = ΔV × (capFld_LH / 总防洪库容)
ΔV_GX = ΔV × (capFld_GX / 总防洪库容)
ΔV_HKC = ΔV × (capFld_HKC / 总防洪库容)
```

典型防洪库容参考：
| 水库 | 防洪库容（亿m³） | 分配比例 |
|------|----------------|----------|
| 陆浑 | 7.44 | 40.2% |
| 故县 | 6.09 | 32.9% |
| 河口村 | 2.31 | 26.9% |

对每个支流水库，重新调用单库模型，增加拦蓄要求（限制更低出库水位，减少出库），重新计算出库过程。

---

#### 3. 迭代收敛判断

重复计算，直到满足以下条件之一：
- ✅ 所有约束都满足
- ✅ 相邻两次计算小浪底最高水位差 < 0.01 m
- ⚠️ 达到最大迭代次数（一般 2-3 次收敛）

---

## 工具调用流程

| 步骤 | 工具 | 说明 |
|------|------|------|
| 1 | `flood-control_get_reservoir_realtime` | 获取五库当前实时水位和蓄水量 |
| 2 | `flood-control_get_reservoir_features` | 获取五库特征参数（汛限水位、防洪库容等） |
| 3 | 新安江模型计算 | 生成各支流水库预报入流过程 |
| 4 | 单库调度计算 | 分别调用单库模型计算三个支流 + 小浪底 |
| 5 | 约束检查与迭代调整 | 若不满足约束，分配额外拦蓄任务，重新计算 |
| 6 | 输出方案 | 生成完整调度方案单 |

## 方案特点

| 特点 | 说明 |
|------|------|
| **符合实际地理流向** | 修正了错误流向，正确体现支流在三门峡下游汇入 |
| **充分考虑三门峡敞泄** | 符合三门峡水库实际运用方式 |
| **复用现有单库模型** | 不需要开发五库耦合模型，降低实现难度 |
| **约束适应性强** | 支持任意水库水位约束，自动向上游分配任务 |
| **与新安江模型兼容** | 新安江提供支流入流，衔接自然 |
| **算法稳定收敛** | 一般 2-3 次迭代即可满足约束 |

## 使用场景

- 现有系统只有单库调度模型，需要生成五库联调方案
- 需要满足特定约束（如小浪底水位不超过某一值）
- 考虑三门峡水库敞泄运用特点
- 基于新安江模型降雨径流预报
