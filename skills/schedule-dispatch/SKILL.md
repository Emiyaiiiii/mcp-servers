# 防洪调度参数修改与调度方案单生成

本技能辅助 Agent 进行防洪调度参数修改和调度方案单生成，通过调用防洪四预 MCP 服务中的两个核心工具完成操作。

## 核心工具

| 工具                      | 用途                         |
| ------------------------- | ---------------------------- |
| `modify_dispatch_param`   | 查看和修改 Dispatch_Par 表中的调度参数 |
| `list_parameter_templates`| 列出所有可用的参数模板       |
| `show_parameter_template` | 展示指定模板的完整参数列表   |
| `apply_parameter_template`| 应用指定参数模板并自动生成调度方案单 |
| `verify_dispatch_result`  | 将新计算结果与模板预期数据进行偏差比对验证 |
| `generate_dispatch_scheme` | 一键生成调度方案单（导入Excel→运行计算→导出结果） |

---

## 一、modify_dispatch_param 工具

### 用途

查看和修改 `data.mdb` 中 `Dispatch_Par` 表的调度参数。支持查看所有参数列表（`action="list"`）和更新指定参数值（`action="update"`）。

### 参数说明

| 参数           | 类型   | 必填 | 说明                                                         |
| -------------- | ------ | ---- | ------------------------------------------------------------ |
| `action`       | string | 是   | 操作类型：`"list"` 查看所有参数，`"update"` 修改指定参数       |
| `station_name` | string | 否   | 站点名称（如 `"小浪底"`、`"三门峡"`、`"花园口"`），精确匹配 `stnm` 字段。`action="list"` 时可不传；`action="update"` 时必传 |
| `param_desc`   | string | 否   | 参数描述关键词（如 `"敞泄"`、`"分洪"`、`"防洪高水位"`），部分匹配 `Instruction` 字段。`action="list"` 时可不传；`action="update"` 时必传 |
| `new_value`    | number | 否   | 新的参数值。`action="update"` 时必传                           |

### 调用示例

**查看所有参数：**
```
modify_dispatch_param(action="list")
```

**查看某个站点的所有参数：**
```
modify_dispatch_param(action="list", station_name="小浪底")
```

**修改参数值：**
```
modify_dispatch_param(action="update", station_name="小浪底", param_desc="敞泄", new_value=0)
```

## 三、show_parameter_template 工具

### 用途
展示指定参数模板的完整参数列表，让用户可以提前查看所有参数设置，确认后再应用。

### 参数说明
| 参数             | 类型    | 必填 | 说明                                                         |
| ---------------- | ------- | ---- | ------------------------------------------------------------ |
| `template_name`  | string  | 是   | 模板名称关键词，支持模糊匹配                                 |

### 调用示例
**查看常规调度模板的完整参数：**
```
show_parameter_template(template_name="常规调度")
```

## 四、apply_parameter_template 工具

### 用途
应用指定名称的参数模板，将模板中预设参数写入 Dispatch_Par 表，可选择自动生成调度方案单。这是快速生成标准洪水调度方案的推荐方式，无需手动逐一修改参数。

### 参数说明
| 参数             | 类型    | 必填 | 说明                                                         |
| ---------------- | ------- | ---- | ------------------------------------------------------------ |
| `template_name`  | string  | 是   | 模板名称关键词，支持模糊匹配（如 `\"方案一\"`、`\"常规调度\"`、`\"上大\"`均可匹配到对应模板） |
| `generate_scheme`| boolean | 否   | 应用参数后是否自动生成调度方案单，默认 `true`                 |

### 调用示例
**应用上大洪水控制方案一，并自动生成方案：**
```
apply_parameter_template(template_name=\"方案一\", generate_scheme=true)
```

**仅应用参数模板，不自动生成方案：**
```
apply_parameter_template(template_name=\"常规调度\", generate_scheme=false)
```

## 五、verify_dispatch_result 工具

### 用途
将新计算出来的调度方案结果，与参数模板中存储的预期数据进行逐点比对，输出偏差验证报告。也可以在仅应用参数后，验证更新后的参数是否与模板参数表完全一致。

### 参数说明
| 参数             | 类型    | 必填 | 说明                                                         |
| ---------------- | ------- | ---- | ------------------------------------------------------------ |
| `template_name`  | string  | 是   | 模板名称关键词，支持模糊匹配                                 |

### 调用示例
```
verify_dispatch_result(template_name="演练洪水-常规调度")
```

---

## 六、generate_dispatch_scheme 工具

### 用途

一键生成调度方案单。工具内部自动完成导入 Excel → 运行计算 → 导出结果的全流程，无需额外参数。五库联调（三门峡、小浪底、陆浑、故县、河口村）直接调用此工具即可。

### 参数说明

无参数。

### 调用示例

```
generate_dispatch_scheme()
```

## 九、参数匹配规则

当 `action="update"` 时，`station_name` 和 `param_desc` 的匹配规则如下：

1. **station_name 精确匹配** `Dispatch_Par` 表的 `stnm` 字段（站点名称）
2. **param_desc 部分匹配** `Instruction` 字段（关键词包含即可），如 `"敞泄"` 可匹配到 "0表示敞泄，1表示控泄"
3. **如果匹配到多条记录**，列出所有匹配项，由用户确认后再修改
4. **如果匹配到 0 条**，返回所有可用参数列表，提示用户重新选择

### 参数模板匹配补充说明
当使用 `apply_parameter_template` 和 `verify_dispatch_result` 时：
1. `template_name` 支持模糊匹配，输入模板名称关键词即可
2. 当存在同名不同分类的模板（如"上大洪水/方案一"和"下大洪水/方案一"都叫"方案一"），建议同时带上分类关键词（如 `"下大洪水方案一"`），可提高匹配准确率，避免匹配到错误分类的模板

## 十、典型控制案例：控制花园口最大流量不超过指定值

### 场景示例
用户需求："控制花园口流量不超过13000"

### 调整步骤

1. **修改核心花园口控制参数**：
   - `小浪底`: "小浪底保滩库容用完后，转控花园口的流量" → 修改为目标流量值（比目标流量低1000~2000m³/s，提前严格控泄）
   - `花园口`: "判别库群退水时刻设置的花园口流量阈值" → 修改为目标流量值
   - `花园口`: "下大洪水，退水过程控制的花园口流量" → 修改为目标流量值
   - `花园口`: "判别支流水库关门时刻的花园口流量" → 修改为目标流量值

2. **调整小浪底关键参数**：
   - **降低初始水位**：尽可能降低小浪底初始水位，比如从250.86降到245.0，额外增加约10亿m³拦洪库容，用于削峰
   - **降低保滩流量**：将小浪底保滩流量从默认的4500逐步降低到3000~3500，提前拦蓄洪水，推迟洪峰到达时间
   - **降低转控判别水位**：将 "4500转控10000的判别水位" 适当降低（比如从254.0降到245.0），让小浪底更早进入严格控泄模式，充分利用库容拦蓄

3. **提高支流水库预泄流量**：
   - 提高陆浑、故县、河口村三个支流水库的预泄流量，让支流水库提前腾出库容，洪水到来时能够更多拦蓄，减少向花园口的下泄量
   - 示例调整：陆浑从200提高到400，故县从200提高到400，河口村从50提高到200

4. **重新生成方案**：
   - 调用 `generate_dispatch_scheme()` 重新计算，查看结果，如果仍未达标可以再按上述方向进一步降低参数值，直到满足要求或达到模型参数极限

### 结果说明
如果调整后花园口最大流量仍超过目标：
- 检查小花间（小浪底至花园口区间）天然来水，如果小花间本身最大来水已经超过目标流量，则即使五库全部拦蓄也无法满足控制要求，这是洪水本身的水情特性决定的
- 需要启用东平湖、北金堤等分洪工程进一步分洪才能满足控制目标
